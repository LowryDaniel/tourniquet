Set up Slack alerts

Slack alerts let Tourniquet message you when your spending is about to hit the cap. You can tap a button in Slack to raise the cap without opening the dashboard.

What you need first

  • A Slack workspace where you can create and install apps (your own workspace works fine)

Basic setup — 2 minutes, alerts only

  1. Go to api.slack.com/apps and click Create New App → From scratch. Name it Tourniquet.
  2. In the left sidebar choose Incoming Webhooks, toggle it On, then click Add New Webhook to Workspace. Pick a channel or DM yourself, then click Allow.
  3. Slack shows you a URL starting with https://hooks.slack.com/services/…. Copy it.
  4. Open ~/.tourniquet/.env and add: 
    SLACK_WEBHOOK_URL=https://hooks.slack.com/services/YOUR/WEBHOOK/URL
  5. Restart Tourniquet (Ctrl+C then tourniquet start), then run the test command: 
    tourniquet test-alerts --key YOUR_KEY_NAME
    You should see a message appear in Slack within a couple of seconds.
Bot tokens (starting xoxb-) and app tokens (starting xapp-) are different things — you need both for in-app one-tap buttons. A webhook URL alone gives you alerts, but tapping the button in Slack will open a browser tab instead of applying the change directly.

Optional: in-app one-tap buttons — 8 more minutes

Without this, Slack buttons open a browser tab to confirm. With this, tapping a button in Slack applies the change immediately, with no browser needed.

  1. In your Slack app settings: Settings → Socket Mode → Enable Socket Mode ON. When prompted, generate an app-level token with scope connections:write. Copy the xapp-… token that appears — Slack shows it only once.
  2. Go to Features → OAuth & Permissions → Bot Token Scopes → Add chat:write. Then click Install to Workspace and approve. Copy the xoxb-… token shown.
  3. In Slack, open the channel you want alerts in and type /invite @Tourniquet. Then right-click the channel name → Copy link. The channel ID is the last part of the URL (e.g. C123ABC456).
  4. Go to Features → Interactivity & Shortcuts → toggle Interactivity ON. Leave the Request URL blank — Tourniquet uses Socket Mode, not a public URL.
  5. Add all three lines to ~/.tourniquet/.env: 
    SLACK_APP_TOKEN=xapp-1-...
SLACK_BOT_TOKEN=xoxb-...
SLACK_CHANNEL_ID=C123ABC456
  6. Restart Tourniquet and watch for Slack Socket connected in the log. Then test: 
    tourniquet test-alerts --key YOUR_KEY_NAME

Set up Telegram alerts

Telegram alerts buzz your phone immediately. Tapping a button in the message applies the change with no browser needed — no public URL required.

What you need first

  • A Telegram account (the free app on your phone or desktop)
  1. Open Telegram and search for @BotFather — make sure it has the blue verified tick. Tap Start, then send /newbot. Follow the prompts to give your bot a name and username (must end in bot). BotFather gives you an API token that looks like 123456789:AAEhBP… — copy it.
  2. Search Telegram for the bot username you just created and tap Start. Send any message (e.g. hi). This lets the bot reply to you.
  3. Find your Telegram chat ID by visiting this URL in your browser (replace TOKEN with the token from step 1): 
    https://api.telegram.org/botTOKEN/getUpdates
    Look for "chat":{"id": in the response. That number is your chat ID.
  4. Add both lines to ~/.tourniquet/.env: 
    TELEGRAM_BOT_TOKEN=123456789:AAEhBPzLhfg-xxxxxxxxxxxxxxxxxxxxxxxxx
TELEGRAM_CHAT_ID=987654321
    Paste just the number for the chat ID — not the label. 987654321 is correct; Chat ID: 987654321 will break it.
  5. Restart Tourniquet and test: 
    tourniquet test-alerts --key YOUR_KEY_NAME
    Your phone should buzz within a second or two.

Set up email alerts

Email alerts require a domain you own (like yourname.com). Gmail addresses won't work. If you don't own a domain, use Slack or Telegram instead — they're simpler and free.

What you need first

  • A domain you own with access to its DNS settings
  • A free account at resend.com
  1. Sign up at resend.com (free tier: 3,000 emails per month).
  2. Go to Domains → Add Domain and enter your domain. Resend shows you DNS records to add. In your DNS provider (Cloudflare, etc.), add those TXT records. Wait 1–5 minutes for verification.
  3. Go to API Keys → Create. Name it tourniquet, permission: Sending access. Copy the key.
  4. Add to ~/.tourniquet/.env: 
    RESEND_API_KEY=re_xxxxxxxxxxxxxxxx
RESEND_FROM_EMAIL=alerts@yourdomain.com
  5. In the Tourniquet dashboard, set each key's alert email field to the address where you want alerts delivered.
  6. Restart Tourniquet and test: 
    tourniquet test-alerts --key YOUR_KEY_NAME
    Check your inbox (and spam folder — new domains often land there first).

Set up a webhook (Zapier, n8n, Home Assistant, custom)

A webhook lets Tourniquet ping any URL when an alert fires. Use this to connect to Zapier, n8n, Home Assistant, Discord, or your own server.

  1. Get a target URL that accepts POST requests with JSON:
    • Zapier: new Zap → trigger "Webhooks by Zapier" → "Catch Hook" → copy URL
    • n8n: new workflow → Webhook node → copy production URL
    • Home Assistant: automation with Webhook trigger
  2. Add to ~/.tourniquet/.env: 
    ALERT_WEBHOOK_URL=https://hooks.zapier.com/hooks/catch/.../...
  3. Restart Tourniquet and test: 
    tourniquet test-alerts --key YOUR_KEY_NAME
    Check your Zapier/n8n/HA history — you should see one incoming POST.
