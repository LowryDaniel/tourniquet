Keep Tourniquet running while this Linux box sleeps

By default many desktop Linux distros suspend after a period of inactivity (especially laptops on battery). Wrap the proxy in systemd-inhibit so the OS keeps it awake while it's running:

systemd-inhibit --what=idle:sleep --who=tourniquet --why="cap enforcement" tourniquet start

Stop the proxy with Ctrl+C and the inhibit lock is released automatically. For a headless server / Pi / VM you usually don't need this — see the 24/7 deployment guide for systemd unit examples.
